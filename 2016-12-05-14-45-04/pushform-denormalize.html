Pushforms - Denormalized

(ID) Name Description
(100998) MS_GRC_AREA_OF_COMPLIANCE Use this form to create Area of Compliance object in the system. Area of Compliance object can be tied to other objects such as Core Objects, risks, functions and other objects.
(100999) MS_GRC_ASSET_CLASS Use this form to create Asset CLass object in the system. Asset Class is a mandatory relation for Asset object.
(101000) MS_GRC_ASSET Use this form to create Asset object in the system.It is always related to an Asset Class Object. Asset object can be tied to other objects such as Control, Risk and other objects
(101001) MS_GRC_CCY_UPLOAD_DETAILS CCY_UPLOAD_DETAILS
(101002) MS_GRC_COMPLIANCE_OBJECT A user with the appropriate access rights can define one or more Compliance Object in the system. For a single Compliance Object created one can tie any number of Core Objects, risks, functions and other objects
(101003) MS_GRC_CONTROL Use this form to create Control object in the system. Control object can be related to any any oyher type of object other than Reference.
(101004) MS_GRC_CORE_OBJECT A user with the appropriate access rights can define one or more Core Object in the system. For a single Core Object created one can tie any number of Risks, Compliance Objects and other objects
(101005) MS_GRC_CURRENCY_UPLOAD CURRENCY_UPLOAD
(101006) MS_GRC_DATA_UPLOAD_FORM This form is used to upload the objects in the system through data upload module. This module is used when bulk data is required to be created in the system.
(101007) MS_GRC_EMAILS_AND_ASSI It is used to generate emails and assignments to the assigned users. It can be in the form of Approvals,Clarification,Reminder,Expiry.
(101008) MS_GRC_EVIDENCE It is used to Evidence objects. Supporting documents can be uploaded using this.
(101009) MS_GRC_EXCEPTION Exceptions are special cases that deviate from the normal behavior in a business process and need to be cared for exceptionally, normally by human intervention. Their cause might include: process deviation, malformed data, infrastructure or connectivity issues, poor quality business rules, etc.
(101010) MS_GRC_FINANCIAL_ACCOUNTS Financial accounts are basic general accounts that are related to the internal finances of the company and they can be tied either to function, Core Object, risk or Compliance Object Usually a company will begin by designing financial Core Objects first, as the SOA legislation is a mandate for improving financial reporting. Hence, the Core Object designed will be applicable to one or more financial accounts. These accounts may be balance sheet and/or income accounts. Financial Accounts are also defined on a company wide basis
(101011) MS_GRC_FUNCTION A user with the appropriate access rights can define one or more functions in the system. For a single function created one can tie any number of Core Objects.
(101012) MS_GRC_KPI_KRI_DATA_ENTRY KPI KRI Data Entry
(101013) MS_GRC_KPI_KRI_DEFINITION <P>Use this form to setup and schedule KPI/KRI Definition for organizational/item specific values related to the BUs or KRIs for Risk Management applications. </P>
(101014) MS_GRC_MODEL_REFERENCE Use this form to create model references in the system. Model Reference object can be tied to other objects such as Core Objects and other objects
(101015) MS_GRC_OBJECTIVES Objectives form the foundation of any GRC activity in the Organization. Implementation of Processes and Controls, identification of Risks, assessing performance etc are all tied to achievement of Objectives.
(101016) MS_GRC_OBJ_RELATIONS Object Relationships
(101017) MS_GRC_OBJ_REL_CONFIG It contains relationshi details for all the objects.
(101018) MS_GRC_ORB_RELATIONS ORB Relations
(101019) MS_GRC_ORG_PROFILE Organizational Profile enables users to pick an existing Organization and append additional information about the Organizations available in the system.
(101020) MS_GRC_PROCESS Process content enables users to define various Business Processes observed within the Organization.
(101021) MS_GRC_PRODUCT Product content form can be used to define all the Products that are developed / manufactured and sold by the enterprise.
(101022) MS_GRC_QUESTION_PROCEDURE A user with the appropriate access rights can define one or more Control Test/Assessment Question in the system.
(101023) MS_GRC_REFERENCE Using this object, reference can be provided either by uploading the document or through HTTP link.
(101024) MS_GRC_REGULATORY_BODY Use this form to create regulatory body object in the system. Regulatory body object can be tied to other Area of Compliance objects.
(101025) MS_GRC_REQUIREMENT Requirement content template can be used to specify various requirements and guidelines laid down by regulatory bodies or agencies that summarize hundreds of laws and regulations. This form is used by NERC Coordinator and Business Administrator to create NERC Requirement within MetricStream application. Each Requirement can be tied to a single Area of Compliance and Standard.
(101026) MS_GRC_RISK Risk is defined as a probability or threat of damage, injury, liability, loss, or any other negative occurrence that is caused by external or internal vulnerabilities, and that may be avoided through preemptive action. This content template can be used to define any kind of Risks, at various levels or hierarchy, related to Objectives, Controls etc and these Risks can be chosen for assessments through Risk Management Solution.
(101027) MS_GRC_STANDARD The Standard Content template is used to define various Standards defined by Regulatory Bodies such as NERC Standards, ISO Standards etc.
