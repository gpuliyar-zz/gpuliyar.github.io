This field is used to delete the request logically. In case any request has been deleted using delete check box option in resource request/allocation report the value in this field will be updated as N. Otherwise the value will be Y

This field is used to capture the SYSTEM Level parameters.

This field is used to capture the System level Configuration Parameters. There is an infolet attached which will get the all System Level Configuration Parameters

Proactive Flag
PROACTIVE_FLAG

This field is used to capture the status of allocation. Value in this field will be Y in case the allocation is made in scheduler proactively. The value in this field will be N in case the request has made from resource request form and later allocated in scheduler.

OPEN_REQUEST
OPEN_REQUEST

This is check box field which is used make a wish list. If check box is checked then it is wish list and Resource Pool owner will have an option to allocate or reject the request. Incase the check box is not selected then resource pool owner will not have an option to reject. He can only allocate resource. However this field is disabled in RSM 6.1 as this feature is out of scope. But in later releases this will be enabled.
