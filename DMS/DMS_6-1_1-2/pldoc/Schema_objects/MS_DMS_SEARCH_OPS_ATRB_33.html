Package MS_DMS_SEARCH_OPS_ATRB_33

======================================================================
Module: MS_DMS_SEARCH_OPS_ATRB
Description: This Package Encapsulates All the modules for Document Search. The search uses the Oracle Text domain Index concept.
======================================================================
____________________________________________________________________________________
PROCEDURE NAME Usage
--------------------- ---------------------------------------------------------------
- SP_DOCUMENT_HOME Result Set For DMS Landing Page.
- SP_DMS_SEARCH Result Set For DMS Search Page.
- SP_DMS_QUERY_DRIVER Logic driven by the Parameter set to determine the Search Method to be used.
- SP_DMS_PROD_DOCS Builds Search Query For Users (No Advance Search).
- SP_DMS_ALL_PROD_DOC Builds Search Query For Users/Admin/Manager (All the Version).
- SP_DMS_DOCMNG_DOC Builds Search Query For Admin/Manager (Advance Search).
- DMS_SEARCH_PLCY Result Set For DMS Policy Page.
- DMS_SEARCH_FAV Result Set For DMS Favourite Page.
- DMS_GET_CAT_SRCH_FLTR This formats CATEGORY input from UI.
- DMS_XTND_ATRB Get all the columns from the table 'MS_DMS_DOC_EXTENDED_ATTRIBS'.
____________________________________________________________________________________

Flow:
1. LANDING_PAGE
______________________________________________________________________________________________________________
|UI(JAVA) |<======> |SP_DOCUMENT_HOME (Fetch Result)| ______________________________________________________________________________________________________________

2. SEARCH_PAGE
______________________________________________________________________________________________________________
|UI(JAVA)|<======> |SP_DMS_SEARCH(Parameter input)(Using methods builds dynamic string/query)|<======>| SP_DMS_QUERY_DRIVER |
(Invokes One Of The Below which builds actual dynamic string/query) )
||
|--------------------------------------|
| SP_DMS_PROD_DOCS |
| OR |
| SP_DMS_ALL_PROD_DOC |
| OR |
| SP_DMS_DOCMNG_DOC |
|--------------------------------------|
______________________________________________________________________________________________________________

3. DEBUG Process
______________________________________________________________________________________________________________

STEP -0: Log as SYSTEMI-->configuration parameter(Tab): MS APPSTUDIO PARAMETER value is set to 0. STEP -1: The Input Parameters are written into the table MS_DMS_SEARCH_OPS_ATRB_lOG.
STEP -2: The Query for specific calls made from UI are written into MS_DMS_SEARCH_OPS_ATRB_lOG based on the input given or select on the UI.
STEP -3: Call the Procedure SP_DMS_SEARCH with parameters logged in the table MS_DMS_SEARCH_OPS_ATRB_lOG.
______________________________________________________________________________________________________________
