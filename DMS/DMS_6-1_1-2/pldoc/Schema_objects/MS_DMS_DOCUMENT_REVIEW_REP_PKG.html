GET_ACTION_DRILLDOWN

public VARCHAR2 GET_ACTION_DRILLDOWN(P_PROC_INS_ID IN NUMBER, 
                                     P_STAGE_ID IN NUMBER, 
                                     P_STG_SEQ IN NUMBER, 
                                     P_OBJ_ID IN NUMBER, 
                                     P_STAT IN NUMBER, 
                                     P_INS_ID IN NUMBER, 
                                     P_REC_NUM IN NUMBER, 
                                     P_LIFECYCLE_ID IN NUMBER)
This is function returns the action value for the Review Assignments.
Parameters:
P_PROC_INS_ID - The Process Instance Id For the Document.
P_STAGE_ID - The Stage in which the assignment is in Pending state.
P_STG_SEQ - The Stage Number in which the assignment is in Pending state.
P_OBJ_ID - The Document Object Id for Which the report should display data.
P_STAT - This Flag will determine whether we are fectching the data for Next stage or Future stage.
P_INS_ID - Instance id for the Review Assignments
P_REC_NUM - RecNum for the Review Assignments
P_LIFECYCLE_ID - The lifecycle Id for which the report should display data
Returns:
Returns action value for the Review Assignments.
Throws:
EXP_MANDATOR_PARAM - If the value for Instanceid,instance rec num,lifecycle id are null rhen then storing the error details in MS_DMS_ERR_LOGS table.
OTHERS - If there is any Other Error then storing the error details in MS_DMS_ERR_LOGS table.

GET_INPROCESS_DOC_DET

public MS_DMS_DOC_REV_DET_TYP GET_INPROCESS_DOC_DET(DOC_NAME IN VARCHAR2 DEFAULT NULL, 
                                                    AUTHOR_NAME IN VARCHAR2 DEFAULT NULL, 
                                                    EFFECTIVE_DATE IN DATE DEFAULT NULL, 
                                                    LIFE_CYCLE_NAME IN VARCHAR2 DEFAULT NULL, 
                                                    DUE_DATE_AFTER IN DATE DEFAULT NULL, 
                                                    DUE_DATE_BEFORE IN DATE DEFAULT NULL, 
                                                    LOGIN_ID IN VARCHAR2)
This is a pipelined function to display the records in Document Review Progress Report Based on filter parameter.
Parameters:
DOC_NAME - The Document Name for Which the report should display data.
AUTHOR_NAME - Author Name for the Document.
EFFECTIVE_DATE - The effective Date of the document.
LIFE_CYCLE_NAME - The lifecycle name for which the report should display data
DUE_DATE_AFTER - Due date is the assignment due date.This is the User selected date after which the report should display data
DUE_DATE_BEFORE - Due date is the assignment due date.This is the User selected date before which the report should display data
LOGIN_ID - The logged in user who is trying to open the Report
Returns:
Returns a pipe row of type MS_DMS_DOC_REV_DET to the Report Level.
Throws:
USER_EXCEP_DATE - When 'Login ID is Null' It will thorow this exception.
OTHERS - If there is any Other Error.
