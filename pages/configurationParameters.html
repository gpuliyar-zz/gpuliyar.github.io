#1 : MS_QSM_Configuration

Scope Enterprise
Description Use this parameter type to configure Survey Management application.

Parameter Name Description List Of Value Value
QSM_Debug The value that appears in the Parameter Value field determines whether to enable or disable the debug code.The possible values that you can enter are: Y and N. Y
IS_FindingsTab_Required Use this parameter to enable the Findings/Issues tab in the survey response form Y
Response_Assignment_Validity_After_Due_Date The value that appears in the Parameter Value field determines the number of days after the due date the Respond To survey/scorecard/certification assignments should be withdrawn from the respondents. 365
Approval_Assignment_Validity_After_Due_Date The value that appears in the Parameter Value field determines the number of days from the due date after which the Respond To survey/scorecard/certification assignments should be withdrawn from the approver. 365
Enable_Context_Field_Updation This parameter would enable the Context Field Data Population while generating a survey response assignment Y
Offline_Survey_Template_Header This parameter allows the user to configure default header in Offline Survey template. Offline Survey
Number_of_Days_after_due The value that appears in the Parameter Value field determines the number of days before the due date the respondent receives reminder email notification to respond to the survey/scorecard/certification. 0
Enable_Auto_Creation_of_Issues This parameter enables or disables auto creation of issues in ISM on crossing the due date of a survey response MS_QSM_Yes_No Yes
Issue_Landing_Stage This parameter indicates the stages in which the issues added from auto creation to be landed MO
Enable Response Pre Population This parameter will determine if the pre population of survey response would be enabled or not in the system Y
Capture_Initial_Response This parameter specified whether the initial survey response to be captured in the survey change history report. Y
Enable Findings or Issues for Questionnaires The value that appears in the Parameter Value field determines whether the Findings tab in the Respond To Survey/Scorecard/Certification form needs to be enabled or not.The possible values that you can enter are: Y and N. Y
Disable_Comments_RTF_in_Response_Form This parameter specifies whether Response Comments are RTF Based or Plan Text. MS_QSM_ENABLE_OR_DISABLE_RTF No

#2 : MS_QSM_Email_Configuration

Scope Enterprise

Parameter Name Description List Of Value Value
Number_of_Days_before_due_date The value that appears in the Parameter Value field determines the number of days prior to which a notification needs to be sent to the respondents before the Response Due Date. 3

#3 : MS_QSM_Performance_Configuration

Scope Enterprise
Description Use this parameter type to configure Survey Management performance-related settings.

Parameter Name Description List Of Value Value
Maximum_Session_Sleep_Time The value that appears in the Parameter Value field indicates the maximum DB sleep time is seconds while controlling the DB sessions to improve the performance 120
Maximum_Allowed_Sessions This parameter specifies the maximum number of allowed sessions in the DB at a particular point of time (Survey Process). 20
Minimum_Session_Sleep_Time The value that appears in the Parameter Value field indicates the minimum DB sleep time is seconds while controlling the DB sessions to improve the performance 60
Enable_Progressive_Submission This flag indicates whether progressive submission has been enabled or not N
Score_Calculation_From_Backend The value that appears in the Parameter Value field determines the score calculation. If you enter Y, then the score is calculated from backend, else the score is calculated from JS N
DB_Sleep_Required This Parameter indicates whether DB Sessions need to be controlled for performance improvement Y
Response_Upload_Thread_Pool_Size The value that appears in the Parameter Value field specifies the number of threads that need to be started for the response upload process 5
